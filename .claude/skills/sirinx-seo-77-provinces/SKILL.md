# SIRINX SEO/AEO 77 Provinces — Strategy v1.0

**Mission:** ครองอันดับ 1 ใน Google สำหรับ "โซลาร์เซลล์ + [จังหวัด]" ทุก 77 จังหวัดในไทย
ด้วย SEO/AEO landing pages ที่ localize สำหรับแต่ละจังหวัด + AI search optimization

**Skill #31 in SIRINX skill suite | Connects to: sirinx-master-gem (Skill #30)**

---

## Section 1: Strategy Overview

### Primary Goal
- **ติดอันดับ #1** สำหรับ keyword "โซลาร์เซลล์[จังหวัด]" และ "ติดตั้งโซลาร์เซลล์[จังหวัด]" ทุก 77 จังหวัด
- **AEO:** ปรากฏใน Google AI Overview, Perplexity AI, ChatGPT Search, Bing Copilot
- **Local SEO:** Google Business Profile integration ทุกจังหวัด
- **Content Cluster:** Province Hub Page → Sub-pages (residential, commercial, factory, farm)
- **Target:** 385 pages total (77 hubs × 5 pages each)

### Why This Works
```
77 จังหวัด × avg 500 searches/เดือน/จังหวัด = 38,500 organic searches/เดือน
CVR 2% = 770 leads/เดือน จาก organic เพียงอย่างเดียว
```

### Competitive Moat
- คู่แข่งส่วนใหญ่ focus กรุงเทพ + ชลบุรี
- SIRINX ครอบคลุม **ทุก** จังหวัด รวมถึง tier-2/tier-3 cities
- HQ ที่พิษณุโลก = authority สำหรับภาคเหนือตอนล่าง + Isan

```typescript
interface ProvinceData {
  id: number;
  slug: string;
  nameTH: string;
  nameEN: string;
  region: 'north' | 'northeast' | 'central' | 'east' | 'west' | 'south';
  priority: 'P1' | 'P2' | 'P3';

  // Solar & Energy Data
  avgIrradiation: number;       // kWh/m²/day (Thailand: 4.5–5.5)
  peakSunHours: number;         // hours/day average
  electricityAuthority: 'MEA' | 'PEA'; // MEA = กรุงเทพ, นนทบุรี, สมุทรปราการ; PEA = ทั่วไป
  avgTariffTHB: number;         // บาท/kWh (PEA ~4.2, MEA ~4.0)

  // Industry & Business
  registeredFactories: number;  // จาก สำนักงานอุตสาหกรรมจังหวัด
  keyIndustries: string[];      // อุตสาหกรรมหลัก
  boiZone: 1 | 2 | 3;          // BOI incentive zone
  gdpRank: number;              // อันดับ GDP ในไทย (1-77)

  // Geography
  distanceFromHQKm: number;     // ระยะทางจากพิษณุโลก (km)
  avgRoofSpaceM2: number;       // พื้นที่หลังคาเฉลี่ย (m²) per building type

  googlePlaceId?: string;       // Google Business Profile Place ID

  serviceArea: string[];        // อำเภอที่ให้บริการ
}
```

## Section 7: Content Generation Rules — Master GEM Integration

### Connection to sirinx-master-gem

เมื่อสร้าง content สำหรับ province page ให้ใช้ **sirinx-master-gem** modes ดังนี้:

```
COMPACT MODE + BRAND-LOCKED + SEO
→ ใช้สำหรับ: Meta descriptions, Title tags, H1/H2 copy
→ Output: ไม่เกิน 160 chars สำหรับ meta, 60 chars สำหรับ title
→ ต้องมี keyword หลักในตำแหน่งแรก

COPY MODE + ENGINEERING-FIRST
→ ใช้สำหรับ: Technical content sections (specs, ROI data, installation details)
→ Output: ภาษาไทยที่อ่านง่าย มีตัวเลขชัดเจน, trust signals
→ ต้องมี proof points: ปี, ตัวเลข, รับประกัน

VISUAL MODE
→ ใช้สำหรับ: Hero image generation per province
→ Image Prompt Template:
   "Professional solar rooftop installation on Thai {BUILDING_TYPE} in {PROVINCE_EN},
    SIRINX brand colors (navy #0A2342 and gold #F5A623), tropical Thai architecture,
    bright sunshine, workers in safety gear, modern industrial setting,
    4K photorealistic, architectural photography style"
→ Building types by province:
   - กรุงเทพ/ชลบุรี/ระยอง → "industrial factory" / "modern warehouse"
   - เชียงใหม่/ภูเก็ต → "resort hotel" / "villa rooftop"
   - ภาคเกษตร → "farm building" / "rice mill"
   - พิษณุโลก (HQ) → "SIRINX headquarters campus"

INFOGRAPHIC MODE
→ ใช้สำหรับ: Province-specific ROI data visualization
→ ต้องแสดง: Irradiation map, ROI timeline, ค่าไฟที่ประหยัดได้/ปี, CO2 ที่ลดได้
→ Brand colors: Navy background, Gold data highlights, Green positive metrics
```

### Content Per Province Checklist
```
□ Hero image (VISUAL MODE) — solar on province-specific building type
□ Meta title (COMPACT + SEO) — ≤60 chars, keyword first
□ Meta description (COMPACT + SEO) — ≤160 chars, CTA included
□ H1 copy — benefit-first, local keyword
□ Province intro paragraph (COPY MODE) — 100-150 words, local context
□ Irradiation infographic (INFOGRAPHIC MODE) — vs Thailand average
□ Industry-specific section (ENGINEERING-FIRST) — province's top 3 industries
□ FAQ (5 standard + 2 province-specific) — AEO optimized
□ ROI calculator default values — province tariff pre-filled
□ LocalBusiness schema — service area = province districts
```

### 9.2 Internal Linking Strategy
```
Hub Page (Province A) MUST link to:
├── All 4 cluster pages of same province
├── 3-5 geographically adjacent provinces (neighbor linking)
├── Same-region province index page (/solar/region/north)
├── Relevant calculator page (/calculator)
└── Contact/Lead page (/contact)

Cluster Pages MUST link to:
├── Hub page of same province (breadcrumb + footer)
├── Same cluster type in adjacent provinces
│   e.g., /solar/chon-buri/factory → /solar/rayong/factory
└── Blog posts with related province content
```

### 9.3 URL Canonical Rules
```
Canonical always points to hub page for provinces:
https://sirinx.com/solar/chiang-mai  (canonical self)

Avoid duplicate content:
- /solar/เชียงใหม่ → 301 redirect → /solar/chiang-mai
- /solar/ChiangMai → 301 redirect → /solar/chiang-mai
- Upper case → 301 to lowercase
```

### 9.4 Core Web Vitals Targets
```
LCP (Largest Contentful Paint):   < 2.5s
FID / INP (Interaction):          < 200ms
CLS (Cumulative Layout Shift):    < 0.1

Optimizations per province page:
□ Hero image: WebP format, width/height specified, lazy-load below fold

□ Schema JSON-LD: In <head>, not body
□ Fonts: Sarabun preloaded, display: swap

□ No render-blocking scripts above fold
```

## Section 11: AEO-Specific Optimization

### What is AEO (Answer Engine Optimization)?
```
AEO = การ optimize content ให้ AI search engines (ไม่ใช่แค่ Google)
สามารถดึงคำตอบไปแสดงใน:

- Perplexity AI answers

- Bing Copilot

```

### AEO Writing Rules for Province Pages
```
1. DIRECT ANSWER FIRST
   ❌ "โซลาร์เซลล์มีประโยชน์มากในยุคปัจจุบันที่ราคาพลังงานสูงขึ้น..."
   ✅ "โซลาร์เซลล์เชียงใหม่ ราคาเริ่มต้น 35,000 บาท/kWp ติดตั้งภายใน 3-7 วัน"

2. INVERTED PYRAMID STRUCTURE
   → ข้อมูลสำคัญ/คำตอบ ก่อน
   → รายละเอียด ตามหลัง
   → Background/context สุดท้าย

3. FACTUAL, CITABLE DATA
   → ระบุปี: "ปี 2026", "ข้อมูล ณ มกราคม 2026"
   → ระบุแหล่งที่มา: "จาก กกพ.", "ข้อมูล Global Solar Atlas"
   → ใช้ตัวเลขที่ชัดเจน: "50-80%" ดีกว่า "มาก"

4. ENTITY OPTIMIZATION (สำหรับ Knowledge Graph)
   → กล่าวถึง SIRINX + Phitsanulok + Solar + Province ใน entity relationships
   → ใช้ structured data ทุก section
   → เชื่อมต่อกับ Wikipedia entities (จังหวัด, PEA, BOI)

5. CONVERSATIONAL QUERIES
   → เขียน FAQ ด้วยภาษาพูด: "ราคาเท่าไหร่", "คืนทุนกี่ปี", "บริษัทไหนดี"
   → ตอบด้วยภาษาพูดเช่นกัน แต่มีข้อมูลครบ
```
